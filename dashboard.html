For IT: How to Build This Dashboard in Amazon QuickSight This tab documents the exact data models, datasets, calculated fields, filters, and visual types needed to replicate every view in this mockup. Full requirements document available separately.

1. Data Sources — What Needs to Be Connected

Data Source 1: Internal Sales Data

INTERNAL Order Management System export (or direct DB connection if available)
Required fields: customer_email, order_id, order_date, order_value, item_count, channel (DS/B2C), consultant_id
Grain: One row per order. Join key: customer_email
Refresh: Nightly via S3 export or Redshift/Athena direct connection to QuickSight

Data Source 2: Adobe Experience Platform / Analytics

ADOBE Campaign event data export from Adobe Experience Platform
Required fields: customer_email, campaign_id, campaign_name, channel, touchpoint_date, utm_source, utm_medium, utm_campaign, utm_content
Grain: One row per campaign touchpoint. Join key: customer_email
Export method: Adobe Data Feed → S3 → AWS Glue / Athena → QuickSight SPICE

Data Source 3: CDP

CDP Unified customer profile export with behavioral signals and identity-resolved email
Required fields: customer_email, first_seen_date, profile_created_date, channel_of_acquisition, identity_confidence_score
Use CDP as identity resolution layer — deduplicated email is the universal join key

2. Data Model — Core Unified Dataset

-- Master Customer Lifecycle Table (built in Athena or Redshift)
SELECT
  c.customer_email,
  c.channel_of_acquisition,
  MIN(o.order_date) AS first_order_date,
  MAX(o.order_date) AS last_order_date,
  COUNT(DISTINCT o.order_id) AS order_count_12mo,
  SUM(o.order_value) AS total_revenue_12mo,
  AVG(o.order_value) AS avg_order_value,
  AVG(o.item_count) AS avg_items_per_order,
  SUM(o.order_value) AS customer_future_value, -- replace with CFV model
  DATEDIFF('day', MAX(o.order_date), CURRENT_DATE) AS days_since_last_order
FROM cdp_profiles c
LEFT JOIN orders o ON c.customer_email = o.customer_email
  AND o.order_date >= DATEADD('year',-1,CURRENT_DATE)
GROUP BY c.customer_email, c.channel_of_acquisition

Stage Assignment Logic (Calculated Field in QuickSight or SQL)

-- Lifecycle Stage (evaluated in priority order)
CASE
  WHEN email IS NULL THEN 'Unknown'
  WHEN order_count_12mo >= 3 AND cfv > 0 THEN 'Loyal'
  WHEN order_count_12mo = 2 AND cfv > 700 THEN 'Loyal'
  WHEN days_since_last_order BETWEEN 181 AND 730 THEN 'Dormant'
  WHEN days_since_last_order > 730 THEN 'Suppressed'
  WHEN days_since_last_order BETWEEN 91 AND 180 AND has_prior_purchase THEN 'At Risk'
  WHEN is_reactive = TRUE THEN 'Reactive'
  WHEN order_count_12mo = 2 AND cfv < 700 THEN 'Repeat Purchase'
  WHEN first_order_within_6mo = TRUE THEN '1st Purchase'
  WHEN email IS NOT NULL THEN 'Known'
  ELSE 'Unknown'
END AS lifecycle_stage

3. QuickSight Datasets Required

QUICKSIGHT DS_CustomerLifecycle — Master unified table (grain: one row per customer, refreshed nightly via SPICE)
QUICKSIGHT DS_OrderHistory — Full order history (grain: one row per order, for trending and cohort analysis)
QUICKSIGHT DS_CampaignTouchpoints — Adobe campaign events joined to customer profiles (for attribution views)
QUICKSIGHT DS_StageHistory — Snapshot table capturing each customer's stage per day (enables migration matrix and movement tracking)
NEW TABLE DS_StageHistory requires a daily snapshot job — recommend AWS Lambda or Glue job at 2AM nightly to write each customer's current stage to a history table

4. QuickSight Dashboard Sheets (Tabs) Required

Sheet 1 — Executive Overview: KPI visuals (Count/Sum/Avg aggregations), Stacked bar (stage distribution), Line chart (loyalty trend), Donut (stage mix), Bar (channel split)
Sheet 2 — Lifecycle Stages: 9× KPI group panels (one per stage), Bar chart (revenue by stage). Use QuickSight "Insight" visual type for auto-generated stage narratives
Sheet 3 — Customer Movement: Pivot table (migration matrix using DS_StageHistory), Two tables (inflows/outflows), Line chart (population trend by stage). Requires parameter for date range selection.
Sheet 4 — Channel & Campaign: Clustered bar (DS vs B2C per stage), Line (acquisition trend by channel), Table (campaign attribution using DS_CampaignTouchpoints joined to DS_CustomerLifecycle)
Sheet 5 — Tracking Link Analyzer: QuickSight parameter + filter control for UTM campaign value. All visuals on this sheet filter to customers whose first touchpoint matches the selected UTM. Requires DS_CampaignTouchpoints as source.

5. Tracking Link Analyzer — Technical Spec

User inputs a utm_campaign value via QuickSight filter control (dropdown or text parameter)
Filter applies to DS_CampaignTouchpoints on utm_campaign field, filtered to first-touch only (touchpoint_rank = 1)
Joined to DS_CustomerLifecycle on customer_email to show current stage of all customers acquired via that campaign
Revenue and order metrics pulled from DS_OrderHistory filtered to same customer cohort
ROI summary requires campaign spend data — recommend adding a DS_CampaignSpend table with campaign_id, spend_amount as a manual upload or CRM feed
-- Tracking Link Attribution Query
SELECT
  o.customer_email,
  c.utm_campaign, c.utm_source, c.utm_medium,
  l.lifecycle_stage AS current_stage,
  COUNT(DISTINCT o.order_id) AS total_orders,
  SUM(o.order_value) AS total_revenue,
  MAX(o.order_date) AS last_order_date
FROM campaign_touchpoints c
JOIN orders o ON c.customer_email = o.customer_email
JOIN customer_lifecycle l ON c.customer_email = l.customer_email
WHERE c.touchpoint_rank = 1 -- first touch only
  AND c.utm_campaign = '{{selected_campaign}}' -- QuickSight parameter
GROUP BY 1,2,3,4,5

6. Access, Security & Export

Dashboard owners (2): Full edit access — Customer Strategy team
Marketing team: Read-only access to Sheets 1–4. Sheet 5 (Tracking) read-only with filter controls enabled
Email export: Configure QuickSight email reports for Executive Overview (weekly, Monday 8AM)
CSV export: Enable on all table visuals — allows team to export customer email lists per stage for campaign targeting
Row-level security: Not required for v1 — all users see all data. Review after B2C channel expansion.
